# Diabetes Neural Network — C Implementation

A single hidden layer neural network built from scratch in pure C, trained on the
Pima Indians Diabetes dataset to predict diabetes from clinical features.
No external ML libraries — just C, math.h, and raw backpropagation.

---

## Project Structure
```
diabetes_nn/
├── src/
│   ├── data.h          # Dataset struct and function declarations
│   ├── data.c          # CSV loading, normalization, shuffle, train/test split

```

## Architecture
```
Input (8 features)
       ↓
Hidden Layer (4 neurons, Leaky ReLU)
       ↓
Output (1 neuron, Sigmoid)
       ↓
Prediction (≥ 0.5 → Diabetic)
```

**Training details:**
- Loss function : Binary Cross Entropy
- Optimizer     : Stochastic Gradient Descent (online, one sample at a time)
- Weight init   : He initialization (suited for ReLU activations)
- Gradient flow : Full backpropagation with two-layer chain rule

## Dataset

Pima Indians Diabetes Database from UCI ML Repository / Kaggle.

- 768 samples, 8 features, binary label (0 = not diabetic, 1 = diabetic)
- Class split: ~65% negative, ~35% positive

Download from:
https://www.kaggle.com/datasets/uciml/pima-indians-diabetes-database

Place the file at:
```
diabetes_nn/data/diabetes.csv
```

## Getting Started

### 1. Clone the repo
```bash
git clone https://github.com/yourusername/diabetes_nn.git
cd diabetes_nn
```

### 2. Download the dataset
Place `diabetes.csv` inside the `data/` folder as shown above.

### 3. Train the model
```bash
make run
```
This trains the network, prints loss and accuracy every 100 epochs,
then prints the confusion matrix, F1 score, and feature importance.
Saves learned weights to `model_weights.bin` automatically.

### 4. Predict on a new patient
```bash
make predict_run
```
Loads saved weights from `model_weights.bin` — no retraining needed.
Enter patient values one by one when prompted.

## Known Limitations

- **Small dataset** — 768 samples is limiting for a neural network
- **Class imbalance** — 65/35 split naturally biases toward negative class
- **Online SGD** — noisy weight updates compared to mini-batch
- **No regularization** — no dropout or L2, slight overfitting possible
- **Single train/test split** — no k-fold cross validation, results vary by shuffle
- **F1 ~0.67 average** — meaningfully better than a dummy classifier (F1=0.0) but not production ready

## What This Project Demonstrates

- Full neural network pipeline in pure C with zero ML libraries
- Manual backpropagation with two-layer chain rule
- He weight initialization and Leaky ReLU to prevent dying neurons
- Gradient clipping to prevent exploding gradients
- Binary Cross Entropy loss with sigmoid output
- Model persistence via binary weight saving and loading
- Proper evaluation metrics beyond accuracy — F1, precision, recall, confusion matrix
