02 Network Architecture

The network takes 8 normalized patient features, passes them through 4 hidden neurons with Leaky ReLU activation, then produces a single probability output via sigmoid. Threshold at 0.5 for binary prediction.

03 Backpropagation

Full two-layer backpropagation implemented manually in C. The elegant part: BCE loss combined with sigmoid gives a clean gradient that avoids chain rule complexity at the output.

Loss
L = −[y·log(ŷ) + (1−y)·log(1−ŷ)]
Output layer gradient — BCE + sigmoid shortcut
dL/dz2 = ŷ − y
dL/dW2[j] = dz2 × hidden[j]
dL/db2 = dz2
Hidden layer gradient — chain rule
dL/dhidden[j] = dz2 × W2[j]
dL/dz1[j] = dh × leakyrelu'(z1[j]) ← 1 if z1>0, else 0.01
dL/dW1[j][k] = dz1 × x[k]
dL/db1[j] = dz1

The key insight: z1[j] (pre-activation) is cached during forward pass so the ReLU derivative can correctly gate gradients during backprop. Without this cache, dead neurons can't be detected.

07 Limitations

Small dataset

768 samples is limiting for a neural network. More data would stabilize training and improve generalization.

Class imbalance

65% negative / 35% positive naturally biases the model toward predicting no diabetes. No resampling was applied.

Online SGD

One weight update per sample produces noisy gradients. Mini-batch averaging would give smoother, more stable convergence.

No regularization

No dropout or L2 penalty — slight overfitting is possible, though the gap between train/test accuracy is small.

Single split

A single train/test split means results vary with shuffle luck. K-fold cross validation would give more reliable estimates.

Initialization sensitivity

Despite He init and Leaky ReLU, ~1 in 10 runs produces a poor initialization. Fixed seed (42) resolves this for demos.
